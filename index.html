About

CD-Agent is a framework where an LLM agent, running inside a Kali-based Docker container, solves professional digital-forensics challenges (PCAPs, disk images, memory dumps, malware samples). Each run produces a summary.json aggregating points, cost, and token usage across 14 scenarios spanning 23 forensic categories.

What each column means

  • Model — the LLM driving the agent (litellm-style identifier).
  • Agent style — harness: codeblock (fenced output), native-tools (provider tool-calling), or claude-sdk (Claude Agent SDK).
  • Weighted accuracy — points awarded divided by max points. The judge grades factual questions by exact-match and analytical questions with an LLM rubric.
  • Points — awarded / max across all 237 questions.
  • Cost — agent-side USD (judge cost tracked separately in the JSON).
  • Steps — total assistant turns across the run.

Only runs with full coverage (an answer for every question in every scenario) are accepted on the public leaderboard; partial runs can still be evaluated locally with cdagent-eval --summary-over-submitted.

Submit a run

  1. Run CD-Agent with full coverage of all 14 scenarios (237 questions) and produce summary.json via cdagent-eval <run_dir>.
  2. Build a submission file: 
    python scripts/build_submission.py path/to/run_dir \
    --run-name my-model-native-tools-v1 \
    --submitter your-handle \
    --notes "optional description"
  3. Validate locally: 
    python scripts/validate_submission.py data/runs/my-model-native-tools-v1.json
  4. Open a PR against XintianPan/leaderboard containing the new data/runs/<name>.json and the updated data/index.json. CI runs the validator; partial-coverage runs are rejected.

Do not hand-edit summary.json. Submissions must come straight out of cdagent-eval so the model, agent_style, and judge_model fields are accurate.
